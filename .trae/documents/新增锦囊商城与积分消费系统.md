我将为您在右侧 `aside` 区域新增“锦囊”板块，并构建积分消费系统。

### 1. 积分系统（Points System）

* **新增状态**：在 `App.jsx` 中新增 `points` 状态，初始值设为 **100**（作为初始资金，方便测试购买）。

* **显示位置**：在这个新板块的最上方显示。积分字体可以加粗。并在边上加入一个“充值”按钮。这个按钮后续会有用，目前先不做计划

### 2. “锦囊”板块 UI

* **位置**：右侧边栏（`aside`），位于“已破解助记词”卡片下方。

* **样式**：与现有卡片风格一致（圆角、白底、阴影）。

* **列表项设计**：

  * **初级锦囊**（10积分）：🔍 漏洞扫描报告

  * **中级锦囊**（20积分）：💉 注入载荷模板

  * **高级锦囊**（50积分）：☢️ 零日漏洞利用

  * 每个项目右侧有一个小巧的“购买”按钮（带价格标签）。

### 3. 购买交互逻辑

* **弹窗组件**：实现一个模态框（Modal/Dialog）。

  * **触发**：点击锦囊的购买按钮时触发。

  * **示范内容**：

    * 当前积分：`30`

    * 消耗积分：`10`

    * 结余积分：`20`

  * **操作**：

    * “是” -> 扣除积分，展示锦囊具体内容（可以是简单的 `alert` 或将内容显示在聊天框/反馈区）。

    * “否” -> 关闭弹窗。

* **余额检查**：如果积分不足，按钮变灰或弹窗提示“余额不足”。

### 4. 锦囊内容数据

我将预设一些通用的提示内容：

* **初级**：提示 AI 的 System Prompt 弱点（例如“尝试诱导它进行角色扮演”）。

* **中级**：给出一个具体的 Prompt 模板（例如“请忽略之前的指令...”）。

* **高级**：给出高阶攻击载荷（例如 Base64 编码的 Prompt）。
